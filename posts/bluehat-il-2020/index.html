Keeping Windows Secure

Keeping windows secure is hard:

  1. 5.7 Million source code files
  2. 1100 PR per day
  3. 3,600 developer commiting to windows
  4. 440 official branches

Windows is evolving: PCs, XBOX, IOT, Hololens, Cloud. If same code runs on different environment, the security model changes. Example: DirectX.

Therefore security must evolve as well -> the process needed to become agile

Strategy:

  • Threat intellegence - where do we want to spend the time? Red teaming, conferences
  • Scale to developers - How can we make 3,600 devs interact with security?

Day 2

Lectures

Intro - Tomer Schwartz & Eric Doerr

Tomer: Security is a team sports. Eric Doerr, GM MSRC & Cloud Security: CVE-2020-0601: “It’s great having the US government sharing security vulns with us for the first time… Sorta :wink:”

Maddie Stone - Where’s Waldo…’s brother’s and sisters

Variant Analysis: Not sustainable to deal with Vulns one by one. Researchers often find more than one vuln at a time -> attacker are too.

Case studies: Android and Chrome.

UAF in the Android Binder driver due to poll handler using a wait queue that is not tied to the lifetime of the file.

Just becuase a vuln was fixed, doesn’t mean it was fixed correctly.

Seems like Fuzzers might be a great solution, and Static Analysis has its place and can work for variant analysis.

Seems like the point is to make sure that after CVEs, we want to check completeness: patching to a proof-of-concept is not sufficient.

CDPWN Layer 2 Zero-day: Ben Seri and Yuval Sarel

What is CDPwn?

Layer 2 (CDP) vulns: 4 RCEs, 1 DoS. Affects a ton of Cisco boxes, phones, and cameras.

History of VLAN-hopping techniques

Double VLAN tagging.

DTP - Cisco proprietary protocol. Used for configuartion. Abuses features in the switch that allows changing from port to trunk port.

Proprietary Layer 2 protocols

There are many Layer 2 protocols. Cisco is the de facto standard.

Research of 1-days leads to 0-days

CDP is enabled by default, used for discovery.

Ben: talking about the CVE: “Improper error handling of malformed data? Sounds like every bug on every protocol ever”.

Looking over the 1-days: most of them are related to the parsing of over-complicated data structures.

0-days: Stack overflows in parsing code + bad memory mitigations. Hey Cisco, the early 2000’s called, they want their Vulns back! 👴🏽🗝
